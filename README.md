# GO！むらっぴ Version 1.6（Version 1.5ベース）

「むらっぴ」と「ぴよっぴ」が、敵を踏みながら村上記念病院を目指すブラウザ向け横スクロールアクションゲームです。
Windows、Android、iPhoneのブラウザで動作し、PWAとしてホーム画面へ追加すればスマートフォン単独で起動できます。

## Version 1.6の変更点

この版はVersion 1.5のゲーム本体、画面デザイン、物理挙動、既存画像、ステージ編集機能を維持したまま、必要箇所だけを追加修正しています。

- むらっぴの走行アニメーションはVersion 1.5の高品質な4コマPNGをそのまま使用
- ステージ編集へ「新規ステージ」「この面を削除」「一時保存（本編に入れない）」を追加
- 各ステージに「ゲームに入れる / 一時保存のみ（下書き）」を設定可能
- 下書きステージは端末内へ保存されるが、通常プレイのステージ一覧には含まれない
- 敵アニメーションは従来どおり、各敵「歩行3コマ + 撃破1コマ」
- むらっぴのジャンプ、ダメージ、死亡、ゴールへ個別のMP3を割り当てられる設定を追加
- 体力0または穴への落下時に、専用画像で上へ跳ねてから落ちる死亡演出を追加
- 死亡時画像は `murappi_defeated.png` として個別に差し替え可能
- Version 1.5で修正した、ぴよっぴの接地歩行と約0.30秒遅れのジャンプを維持
- PWAキャッシュをVersion 1.6用へ更新

## 操作

- タップ、クリック、Spaceキー、上矢印キー：ジャンプ
- 空中でもう一度タップ：2段ジャンプ
- 敵の上から接触：踏みつけ
- 敵の横や下から接触：体力を1失う
- 体力が0：残機を1失い、チェックポイントから再開
- 残機が0：ゲームオーバー
- 画面上部の `⌂`：現在のプレイを終了してタイトルへ戻る
- 画面上部の `Ⅱ`：一時停止
- 画面上部の `♪`：サウンド切り替え

穴へ落ちても、ジャンプ回数が残っていれば空中から脱出できます。むらっぴの全身が画面下へ消えた時点で落下ミスになります。

## ステージ編集

タイトル画面の「ステージ編集」を選択します。

1. 「編集する面」からステージを選びます。
2. 左側のパレットから配置するパターンを選びます。
3. マップ上をタップ、クリック、ドラッグして配置します。
4. 「全ステージを保存」を押します。
5. 「この面をテスト」で編集中のステージを確認します。

### ステージの新規作成・削除・一時保存

- 「新規ステージ」：草地とスタート・ゴールだけを配置した新しい下書きステージを追加します。
- 「この面を削除」：選択中のステージを削除します。最後の1面は削除できません。
- 「一時保存（本編に入れない）」：マップを端末内へ保存しますが、通常プレイには出しません。
- 「ゲーム本編への反映」を「ゲームに入れる」へ変更して「全ステージを保存」すると、通常プレイへ追加されます。
- ステージ選択欄では、一時保存中の面に `［下書き］` と表示します。

### パターンの追加

1. ステージ編集画面の「パターンを追加・編集」を開きます。
2. 「新規追加」を押します。
3. パターン名と分類を入力します。
4. 当たり判定を選択します。
5. 「画像を選択」からPNG、WebP、JPEG画像を選びます。
6. 「設定を保存」を押します。

画像は端末保存用に最大512ピクセルへ自動的に縮小・最適化されます。透過画像を使う場合はPNGまたはWebPを推奨します。

### 標準パターンの編集

パレットで編集したい標準パターンを選択し、「パターンを追加・編集」を開きます。画像、名称、分類、当たり判定を変更して「設定を保存」を押します。

### パターンの削除

削除対象をパレットで選択し、「削除」を押します。削除したパターンは、全ステージに配置済みのマスからも消去されます。

「標準パターンを復元」を押すと、追加・変更・削除したパターン設定を破棄して初期状態へ戻します。追加パターンを配置していたマスは空白になります。

### 当たり判定

| 設定 | 動作 |
|---|---|
| 通過する | 背景や装飾。むらっぴ、敵とも通り抜ける |
| 上に乗れる | 上面だけ着地可能。横や下からは通過する |
| 上に乗れる・横からぶつかる | 地面や壁。上下左右の衝突判定を持つ |
| 足場なし・落下する | 画像は表示するが足場判定を持たない。画面外へ落ちるとミス |
| 接触するとダメージ | 接触時にダメージを受ける危険地形 |

### 地面の一括配置

「地面を一括配置」は、現在選択中のパターンが「上に乗れる・横からぶつかる」の場合、そのパターンを最下段へ並べます。それ以外を選択している場合は、標準の草地面を使用します。

## マップ一式の書き出し・読み込み

「マップ一式を書き出し」を押すと、次の内容を1つのJSONファイルへ保存します。

- 全ステージのマップ
- 追加パターン
- 標準パターンへの変更
- パターン画像データ

別端末では「マップ一式を読み込み」から同じJSONを選択し、その後「全ステージを保存」を押してください。

旧バージョンのステージ配列だけを含むJSONも読み込めます。ただし、その形式には追加パターン画像は含まれません。

## スマートフォン単独で使う方法

このゲームは、HTTPSで公開したPWAをスマートフォンへ一度追加し、その後はホーム画面から単独起動する方式です。プレイ時にWindows PCを起動しておく必要はありません。

### Android

1. ChromeでゲームのHTTPS URLを開きます。
2. タイトル画面の「スマホに追加」を押します。
3. 「インストール」を押します。
4. ホーム画面の「GO！むらっぴ」から起動します。

### iPhone・iPad

1. SafariでゲームのHTTPS URLを開きます。
2. 共有ボタンを押します。
3. 「ホーム画面に追加」を選びます。
4. ホーム画面の「GO！むらっぴ」から起動します。

一度ゲームと素材を読み込めば、通信できない状態でも起動できます。追加したマップパターンも端末内に保持されます。

### 制約

ZIPをスマートフォンへ保存して `index.html` を直接開くだけでは、iPhoneでPWA、Service Worker、オフラインキャッシュを安定して利用できません。最初の1回はHTTPS対応のWebサーバーへ配置してください。

初回からWeb公開を一切使わず、完全に端末内だけで配布する場合は、Android APKまたはiOSアプリへの変換が必要です。

## 敵PNGの差し替え

敵画像は `assets/enemies/` にあります。

```text
moko_walk_1.png
moko_walk_2.png
moko_walk_3.png
moko_defeated.png
```

設定は `js/config.js` の `assets.enemies` にあります。`walk` 配列へPNGを追加すると、その枚数で歩行アニメーションします。

## むらっぴ画像の差し替え

`js/config.js` の `assets.murappi` を変更します。

```javascript
murappi: {
  idle: "assets/characters/murappi_threequarter.png",
  run: [
    "assets/characters/murappi_run_1.png",
    "assets/characters/murappi_run_2.png",
    "assets/characters/murappi_run_3.png",
    "assets/characters/murappi_run_4.png"
  ],
  jump: "assets/characters/murappi_jump.png",
  hurt: "assets/characters/murappi_hurt.png",
  defeated: "assets/characters/murappi_defeated.png"
}
```

## 死亡時画像の差し替え

死亡演出では `assets/characters/murappi_defeated.png` を使用します。画像ファイルを同名で差し替えるか、`js/config.js` の `assets.murappi.defeated` を変更してください。

死亡演出は、体力が0になった場合と穴へ完全に落下した場合の両方で実行されます。むらっぴが上へ跳ね、回転しながら画面下へ落ちた後で残機を1失います。

## BGM・効果音の差し替え

`js/config.js` の `assets.audio` に音声ファイルのパスを指定します。空文字の場合はWeb Audioで自動生成します。

```javascript
audio: {
  bgm: "assets/audio/bgm.mp3",
  jump: "assets/audio/jump.mp3",
  stomp: "assets/audio/stomp.mp3",
  damage: "assets/audio/damage.mp3",
  death: "assets/audio/death.mp3",
  goal: "assets/audio/goal.mp3",
  hurt: "",       // 旧設定互換
  clear: "",      // 旧設定互換
  gameOver: "assets/audio/gameover.mp3"
}
```

## 保存場所

- 最高スコア、ランキング、ステージ：ブラウザのLocal Storage
- 追加パターン、パターン画像、標準パターンへの変更：IndexedDB

ブラウザのサイトデータを削除すると、これらも削除されます。重要なマップは「マップ一式を書き出し」でバックアップしてください。

## 起動確認

開発用にWindowsで確認する場合は `start_server.bat` を実行し、表示されたURLをブラウザで開きます。

正式なスマートフォン運用では、HTTPS対応の静的Webサーバーへフォルダー一式を配置してください。
