# GO！むらっぴ Version 1.6.4 素材仕様一覧

この文書は、現在のゲームで「どのタイミングに、どの画像・音声を割り当てられるか」と、その標準ファイル名・配置場所を整理したものです。

## 1. 基本ルール

- 画像は原則として **PNG形式・背景透過** を使用します。
- 既存画像を差し替える場合は、同じファイル名で上書きすれば反映されます。
- ファイル名や配置先を変更する場合は、`js/config.js` のパスも変更します。
- 音声は **MP3形式** を標準とします。
- MP3が存在しない、または読み込みに失敗した場合は、内蔵のWeb Audio効果音・BGMへ自動的に戻ります。
- Version 1.6.4ではオンライン時にGitHub Pages上の画像・音声を優先して取得します。更新後はページを再読み込みし、ホーム画面版アプリも再起動してください。

### 踏みつけ画像の推奨仕様

```text
配置先：assets/characters/murappi_stomp.png
形式　：PNG（背景透過）
コマ数：1コマ
推奨　：ジャンプ画像と近い縦横比、足元が画像下端付近
```

`murappi_stomp.png` が存在しない間は、ゲームが自動的に `murappi_jump.png` を使用します。したがって、画像を準備する前でもゲームは壊れません。

## 5. ステージ配置アイテム

配置場所：`assets/items/`

| アイテム | 標準ファイル名 | 効果 | 標準有効時間 | 画像未配置時 |
|---|---|---|---|---|
| ハート | `item_heart.png` | 体力を1回復。上限3 | 即時 | 内蔵ハート型を表示 |
| 残機UP | `item_life.png` | 残機を1増加。上限99 | 即時 | 内蔵の `1UP` 表示 |
| 空中ジャンプ強化 | `item_air_jump.png` | 空中ジャンプを1回から2回へ増加。地上ジャンプを含め合計3回 | 10秒 | 内蔵の `J+` 表示 |

### 推奨画像仕様

```text
形式　：PNG（背景透過）
推奨比：1:1の正方形
推奨例：256×256 または 512×512
余白　：画像外周に約8～12%
向き　：正面向き。足元位置の統一は不要
```

ステージ編集では「アイテム」分類に表示されます。各アイテムは複数配置できます。取得するとそのステージ内では消え、残機を失って復帰しても再出現しません。ステージを最初から開始すると再出現します。

空中ジャンプ強化中はHUDに `JUMP+ 10s` の形式で残り時間が表示されます。

## 6. 効果音・BGM

配置場所：`assets/audio/`

| 発生タイミング | 標準ファイル名 | 動作 |
|---|---|---|
| ステージ開始からプレイ中 | `bgm.mp3` | ループ再生。死亡、ステージクリア、タイトル復帰で停止 |
| 1段ジャンプ・2段ジャンプ | `jump.mp3` | ジャンプ入力が成立した瞬間 |
| 敵の踏みつけ成功 | `stomp.mp3` | 踏みつけ判定成立時 |
| 敵・危険地形からダメージ | `damage.mp3` | 体力が1減った瞬間 |
| 体力0・穴への落下による死亡 | `death.mp3` | 死亡演出開始時 |
| ステージクリア・全面クリア | `goal.mp3` | ゴール判定成立時 |
| ハート取得 | `item_heart.mp3` | ハート取得時 |
| 残機UP取得 | `item_life.mp3` | 残機UP取得時 |
| 空中ジャンプ強化取得 | `item_air_jump.mp3` | 空中ジャンプ強化取得時 |
| 直接ゲームオーバー処理 | `gameover.mp3` | 互換・予備用途。通常の最終残機喪失では `death.mp3` が主に鳴る |

### 標準配置例

```text
assets/
└─ audio/
   ├─ bgm.mp3
   ├─ jump.mp3
   ├─ stomp.mp3
   ├─ damage.mp3
   ├─ death.mp3
   ├─ goal.mp3
   ├─ item_heart.mp3
   ├─ item_life.mp3
   ├─ item_air_jump.mp3
   └─ gameover.mp3
```

ファイルが存在しない場合は、内蔵音へ自動フォールバックします。

### ステージごとに別BGMを使う場合

各ステージデータに `bgm` を指定できます。

```json
{
  "title": "第1ステージ",
  "bgm": "assets/audio/stage1.mp3"
}
```

例：

```text
assets/audio/stage1.mp3
assets/audio/stage2.mp3
assets/audio/stage3.mp3
```

ステージ側に `bgm` が指定されていない場合は、共通の `assets/audio/bgm.mp3` を使います。

## 9. 表示優先順位

同時に複数の状態が成立した場合、むらっぴの画像は次の優先順位で決まります。

1. 死亡：`murappi_defeated.png`
2. ダメージ：`murappi_hurt.png`
3. 踏みつけ直後：`murappi_stomp.png`
4. 空中：`murappi_jump.png`
5. 地上走行：`murappi_run_1.png` ～ `murappi_run_4.png`

この順番により、踏みつけ後に反発ジャンプしても、最初の約0.16秒は踏みつけ専用画像が表示され、その後ジャンプ画像へ切り替わります。
